# (C# 코딩) 파일 비교툴(FileCompare)

## 개요
-C# 프로그래밍학습  
-1줄소개: 파일을 비교하는 앱  

-사용한플랫폼:C#, .NET Windows Forms, Visual Studio, GitHub  

-사용한컨트롤:SplitContainer, Panel, ListView, button, textbox, Label

-구현한기능  
파일 비교 및 상태 표시  
선택 파일 복사 및 덮어쓰기 처리 가능  
파일 비교 결과를 색상으로 표시  
선택한 파일 강조  
(>>>, <<<) 버튼 클릭시 선택한 방향으로 파일 복사 가능



## 실행화면(과제1)
-과제1코드의실행스크린샷

![과제1 실행화면](img/capture7-1.png)


-과제내용  
사용자가 두 폴더의 파일을 직관적으로 비교하고 관리할 수 있도록 기본적인 인터페이스(UI)를 설계하고, 창 크기 변화에 대응하는 유연한 레이아웃을 구축

-구현내용과 기능설명  
메인 타이틀: Label 컨트롤을 사용하여 앱 이름("File Compare")을 상단에 표시  
경로 입력창: TextBox 2개를 배치하여 선택된 폴더의 절대 경로가 표시되도록 설정   
폴더 선택 버튼: 각 텍스트박스 옆에 Button을 배치하여 폴더 브라우저 창을 띄울  준비를 함.  
중앙 기능 버튼: 리스트뷰 사이에 >>>, <<< 버튼을 배치하여 향후 복사 기능을 실행할 공간 확보.  
ListView: Top, Bottom, Left, Right 네 방향 앵커를 모두 설정하여 창과 함께 커지도록 구현.  
우측 버튼 및 텍스트박스: Top, Right 앵커를 설정하여 창이 가로로 넓어질 때 우측 정렬을 유지하도록 구현.  

## 실행화면(과제2)
-과제2코드의실행스크린샷
![과제2 실행화면](img/capture7-2.png)

-과제내용  
사용자가 선택한 두 폴더의 내용을 읽어와 리스트뷰에 표시하고, 파일명과 수정 시간을 비교하여 동일, 최신(New), 이전(Old), 단독 존재 상태를 판별해 서로 다른 색상으로 출력

-구현내용과기능설명  
폴더 선택 기능: FolderBrowserDialog를 사용하여 왼쪽과 오른쪽 폴더 경로를 각각 가져옴  
리스트뷰 설정: View = Details 모드에서 '이름', '크기', '수정일' 컬럼을 구성  
검은색 (Black): 양쪽 폴더에 파일이 모두 있고, 수정 시간도 동일한 경우  
빨간색 (Red): 반대편 폴더의 파일보다 수정 시간이 더 최신인 경우 (New)  
회색 (Gray): 반대편 폴더의 파일보다 수정 시간이 오래된 경우 (Old)  
보라색 (Purple): 한쪽 폴더에만 존재하고 반대편에는 없는 경우 (단독 파일)

## 실행화면(과제3)
-과제3코드의실행스크린샷
![과제3 실행화면](img/capture7-3.png)
![과제3 실행화면](img/capture7-4.png)



-과제내용  
사용자가 ListView에서 선택한 파일을 반대편 폴더로 복사하는 기능을 구현합니다. 이때 파일의 경로를 정확히 파악하고, 기존 파일이 있을 경우 덮어쓰기 여부를 처리하며, 복사 후에는 리스트뷰를 갱신하여 상태(색상) 변화가 나타남  

-구현내용과기능설명  
파일 복사 실행: System.IO.File.Copy(sourcePath, destPath, true) 메서드를 사용하여 파일을 복사  
경로 조합: 선택된 폴더 경로와 파일 이름을 합쳐 전체 경로(Full Path)를 만듬   Path.Combine() 함수를 사용하면 안전하게 경로를 합침

## 실행화면(과제4)
-과제4코드의실행스크린샷
![과제4 실행화면](img/capture7-5.png)


-과제내용  
 기본적인 파일 비교 기능은 유지하되, 선택한 폴더 내부에 또 다른 폴더(하위 폴더)가 있을 경우 그 안으로 파고 들어가 모든 파일을 리스트업하고 비교 대상에 포함시키는 기능을 구현  

-구현내용과기능설명  
SearchOption 활용: Directory.GetFiles나 EnumerateFiles 사용 시 SearchOption.AllDirectories 옵션을 주어 한 번에 모든 하위 파일을 가져오도록 수정  
상대 경로 계산: 하위 폴더의 파일을 비교할 때는 '절대 경로'가 다르더라도 '상대적 위치(폴더 구조)'가 같으면 동일한 파일로 인식하게끔 경로 문자열을 처리  
하위 폴더 포함 파일 비교,"폴더 구조 + 파일 이름"이 일치하는지 확인  
사용자가 폴더를 선택하면 그 아래에 100개의 하위 폴더가 있더라도 빠짐없이 모든 파일을 찾아냄
